The Problem with AI Memory

Today's AI agents forget everything between sessions. Your users repeat themselves. Your agents lose context. Memorose fixes this.

Without Memorose

  • ✗Agents forget user preferences every session
  • ✗No long-term knowledge accumulation
  • ✗Context window stuffing wastes tokens
  • ✗No multi-agent memory sharing
  • ✗Vendor lock-in with cloud-only solutions

With Memorose

  • ✓Persistent memories across sessions and agents
  • ✓Knowledge graph builds over time automatically
  • ✓Hybrid search retrieves exactly what's needed
  • ✓Multi-tenant isolation for SaaS platforms
  • ✓Self-hosted, open-source, your data stays yours

Tiered Storage Architecture

Memories flow from hot cache to warm index to cold storage. Each tier optimized for its access pattern.

L0

Hot Cache

In-memory LRU for sub-ms reads

LRU CacheRecent queriesActive sessions
L1

Warm Index

LanceDB vectors + Tantivy full-text

Vector embeddingsBM25 indexKnowledge graph
L2

Cold Storage

Compressed archive with bitemporal versioning

Bitemporal historyCompressed dataRaft-replicated

5 Lines to Get Started

Add persistent memory to your AI agent in minutes, not days.

quickstart.py
1from memorose import Memorose
2

3client = Memorose("http://localhost:3000")
4

5# Store a memory
6client.add(
7    content="User prefers dark mode and Python",
8    user_id="alice",
9    app_id="my-app",
10    metadata={"source": "preferences"}
11)
12

13# Retrieve relevant memories
14results = client.search(
15    query="What does the user prefer?",
16    user_id="alice",
17    limit=5
18)
19

20for memory in results:
21    print(f"{memory.content} (score: {memory.score:.2f})")

pip install memorose · Works with any LLM framework

Everything You Need

A complete memory infrastructure for production AI applications.

Hybrid Search

Core

Combine vector similarity, full-text BM25, and knowledge graph traversal in a single query. Get the most relevant memories every time.

Raft Consensus

HA

Distributed multi-node replication with Raft ensures your memory layer survives node failures without data loss.

Multi-Tenant Isolation

First-class user_id and app_id scoping. Each tenant's data is fully isolated with row-level filtering.

Knowledge Graph

New

Automatically extract entities and relationships from memories. Traverse connections your agents can reason about.

Active Forgetting

Bitemporal architecture with configurable TTLs and importance-based decay. Old memories fade, key ones persist.

Built-in Dashboard

Real-time visualization of memories, graphs, agent activity, and cluster health. No separate tooling needed.
