# Deployment

Production deployment guides for Memorose.

## Docker Compose (Recommended)

Create a `docker-compose.yml`:

```yaml
version: "3.8"
services:
  memorose:
    image: ghcr.io/memorose/memorose:latest
    ports:
      - "3000:3000"
    volumes:
      - memorose_data:/data
      - ./config.toml:/etc/memorose/config.toml
    environment:
      - MEMOROSE_API_KEY=your-secret-key
    restart: unless-stopped

volumes:
  memorose_data:
```

```bash
docker compose up -d
```

## Single Binary

Download the latest release for your platform:

```bash
# Linux (x86_64)
curl -fsSL https://github.com/memorose/memorose/releases/latest/download/memorose-linux-amd64 -o memorose
chmod +x memorose

# macOS (Apple Silicon)
curl -fsSL https://github.com/memorose/memorose/releases/latest/download/memorose-darwin-arm64 -o memorose
chmod +x memorose
```

Run with a config file:

```bash
./memorose --config config.toml
```

## Multi-Node Cluster

For high availability, deploy a 3-node Raft cluster:

```toml
[server]
host = "0.0.0.0"
port = 3000
node_id = 1

[cluster]
peers = [
  "node1:4000",
  "node2:4000",
  "node3:4000"
]
```

Repeat for each node with a unique `node_id`. Raft will automatically elect a leader and begin replicating writes.

## Reverse Proxy (nginx)

```nginx
upstream memorose {
    server 127.0.0.1:3000;
}

server {
    listen 443 ssl;
    server_name memorose.example.com;

    location / {
        proxy_pass http://memorose;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## Monitoring

Memorose exposes Prometheus metrics at `/metrics`. Key metrics:

- `memorose_search_duration_seconds` — search latency histogram
- `memorose_memories_total` — total stored memories
- `memorose_raft_state` — current Raft node state (leader/follower/candidate)

## Backups

```bash
# Snapshot current state
curl -X POST http://localhost:3000/admin/snapshot

# Data is stored in the configured data_dir
# Back up the entire directory for full recovery
```
